Problem Description

- Existe un sistema con chats especializados por tipo (maitre, bi, brain, consultas, etc.).
- El usuario puede crear varias instancias de un mismo tipo y mantener múltiples conversaciones de ese tipo en paralelo.
- Se requiere persistir el historial completo de cada conversación.
- El historial debe incluir solo lo que el usuario ve, sin detalles internos del sistema.
- La relación de propiedad es: empresa -> oficina -> usuario -> chat.
- Los mensajes tienen timestamp y un orden estable por secuencia.
- El historial debe ser inmutable.
- No hay anonimización por ahora.
- Los chats pueden borrarse, pero no se eliminan por completo de la persistencia.
- Se espera un volumen aproximado de ~300 mensajes por chat; el tamaño total es indefinido.
- No hay búsqueda hoy, pero es un requisito potencial futuro.
- Se pueden enviar textos y archivos (audio, video, imagenes, otros).
- El titulo del chat es editable.
- Idioma: se debe registrar el idioma principal del chat.
- El versionado es deseable, aunque no requerido en la primera version.
- Estados del chat: activo, archivado, borrado.
